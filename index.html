Pillar 1

Documentation

While less glamorous than livestreams or content, documentation is a top priority. It's the front door to Entire and the first thing people go to when they want to use the product. Good docs can make or break adoption.

📦 Updated with each release
✓ Covers all features
🗂 Organized by workflow type
Single agent Claude Code, Cursor, Gemini CLI, OpenCode, FactoryAI. Users maybe using MCP
Multi-agent (2-5) Multiple agents working on related tasks. Users maybe using Beads
Orchestration (10+) Gastown, Ralph Wiggum loops. How Entire tracks activity and decisions at scale

Docs (Agent Era)

AI-Native, Agent-Ready Documentation

Humans read docs, but increasingly, agents and LLMs are the ones finding answers. Make documentation easy to retrieve, quote, and compile into prompts, so people get correct answers even when they never open the page.

AI-native docs Design for LLM consumption and retrieval, not just human browsing. Agent-ready docs Make docs easy for tools and agents to navigate and use as source-of-truth.
→ Make it easy to copy/paste content into AI chatbots as Markdown.
→ Support viewing pages as Markdown (for example, add .md to the URL).
→ Serve an llms.txt file as a directory of Markdown files.

Field Strategy

California + East Coast Coverage

One DevRel in California for fast ecosystem response. One on the East Coast for coverage and different networks. Both converge for major conferences.

California-based DevRel

  • Jump on short-notice meetups and partner moments
  • Run a repeatable Bay Area cadence (builder night + workshops)
  • Broader than SF: optimize for California talent + density

East Coast DevRel

  • Time zone coverage + different community graph
  • More touchpoints with NY/BOS/DC tech circles
  • Parallel cadence: meetups + workshops + office hours
Key idea: turn our field playbook into a template ambassadors can copy locally.

Ambassadors

From Contributors to Meetup Hosts

We identify real builders, spotlight them, then equip them to represent Entire locally through a formal application process.

1 Identify
Code contributors, #share-your-work builders, consistent community helpers.
2 Spotlight
Livestream showcases where they demo what they built with Entire.
3 Apply
Open application form. We review for fit, activity, and regional coverage.
4 Activate
Run local meetups, speak at conferences, write content, host workshops.
Requirements
  • Active Entire user with a public project or contribution
  • History of community participation (Discord, GitHub, socials)
  • Willingness to host or speak at least once per quarter
  • Located in a city/region where Entire needs presence
What ambassadors get
  • Event budget + sponsorship for local meetups
  • Swag, slide templates, workshop repos, and a meetup kit
  • Direct line to the Entire team + early access to features
  • Co-promotion on Entire socials and ambassador spotlight page
  • Conference speaking support + introductions
